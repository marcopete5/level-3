React Badge Form

Assignment

Objective

Create a React website where users can input and submit badge information. Each submitted badge will be displayed in a list below the form. This assignment will help you practice handling forms, state management, and dynamic rendering in React.

Handling forms in React requires controlled components, where input values are tied to state variables. This ensures React manages the form’s state, enabling real-time validation and updates. For example, a controlled input syncs its value with state via `value` and `onChange` props, providing a single source of truth for form data.

Requirements

1. Form Setup

  • Create a form with the following input fields:
    • Text Input: For the badge user’s name.
    • Textarea: For the badge's description or other relevant details.
    • Number Input: For a phone number (must be digits only, no special characters).
    • Radio Buttons: For selecting a badge type (e.g., “Employee”, “Guest”).
    • Checkbox: To agree to terms or conditions.
  • Include a Submit button that:
    • Is disabled if any field is empty.
    • Only enables if all fields meet the validation criteria.

Controlled components are essential for form handling. A common pitfall is using uncontrolled inputs, which can lead to unpredictable behavior. For example: 

<input type="text" name="name" />
This lacks state binding. Correct approach: 
<input type="text" name="name" value={name} onChange={(e) => setName(e.target.value)} />
This ensures React controls the input, enabling validation and state updates.

2. Validation

  • Ensure each input field has a minimum character count of 3.

Real-time validation improves user experience by providing instant feedback. A pitfall is validating only on submit, which can frustrate users. For the phone number, use a regex like `/^\d+$/` to enforce digits-only input. Incorrect: 

if (phone.length >= 3) // Misses non-digit check
Correct: 
if (phone.length >= 3 && /^\d+$/.test(phone))
This ensures robust validation for all inputs.

3. Form Submission

  • Upon submitting the form:
    • Clear the input fields.
    • Add the new badge information to a list displayed below the form.
  • Each submitted badge should appear in a list with all previously submitted badges.

Resetting form inputs after submission requires setting state to initial values. A common mistake is incomplete resets, leaving stale data. Use a single state object for all inputs: 

setFormData({ name: '', description: '', phone: '', type: 'Employee', agreed: false })
This ensures all fields, including radio buttons and checkboxes, reset correctly.

4. Display Badges

  • Render each badge as a card or similar component.
  • Alternate the background color of each badge’s heading (e.g., Red, Blue, Red, Blue).

Use `map` to render dynamic lists, with the index determining alternating colors (e.g., `index % 2 === 0 ? '#ef4444' : '#2563eb'`). A pitfall is using indices as keys without ensuring uniqueness, which can cause rendering issues. Correct: 

{badges.map((badge, index) => <Badge key={index} badge={badge} index={index} />)}
Ensure keys are stable to avoid React re-rendering problems.

Game Plan

1. Pseudocode

  • Define state variables for input values and the list of badges.
  • Create functions for form validation and handling form submissions.
  • Create components for the form and the badge display.

2. Components Needed

  • BadgeForm: A form component with input fields and submit button.
  • BadgeList: A component to render the list of badges.
  • Badge: A component to display each individual badge.

3. State Values

  • Form Inputs: To hold current values of name, description, phone number, badge type, and checkbox.
  • Badges Array: To store the list of submitted badges.

By completing this assignment, you will gain experience with form handling, state management, validation, and dynamic rendering in React.

As a practical example, consider a badge card component: 


function Badge({ badge, index }) {
    const headerColor = index % 2 === 0 ? '#ef4444' : '#2563eb';
    return (
        <div class="section">
            <h3 style={{ backgroundColor: headerColor, color: '#ffffff', padding: '10px', borderRadius: '6px 6px 0 0' }}>
                Badge {index + 1}: {badge.name}
            </h3>
            <p><strong>Description:</strong> {badge.description}</p>
            <p><strong>Type:</strong> {badge.type}</p>
        </div>
    );
}
This component demonstrates modular design with alternating header colors. Move inline styles to CSS classes in production for better maintainability.
