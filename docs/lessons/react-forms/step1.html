Importance of Forms in Web Applications

Forms are essential for any web application. They allow users to interact with your app by submitting data, like filling out registration forms, logging in, or searching for information. In traditional HTML, handling user input can be tricky, often requiring multiple steps to capture and process data.

Simplifying Form Handling with React

But with React, form handling becomes much simpler! React lets us manage form state and user input more efficiently using its powerful state management and event handling features.

React Forms and State

Now, let’s talk about the relationship between React forms and state. In React, state is like a container that holds the data that can change in your app. Whenever the state changes, React re-renders the component to reflect these changes.

For forms, state is crucial. Each input field in a form can be connected to a piece of state. This is called controlled components. The input fields get their values from the state and update the state whenever they change. This creates a single source of truth for form data, making the management of user input more predictable and easier to debug.

Creating a Form Component

Instead of hardcoding a form inside App.jsx, we can make it reusable by creating a Form component. Let's create a form with three fields: title, rating, and description.

Example Code: App.jsx


import React from 'react';
import Navbar from './Navbar';
import RandomMovie from './RandomMovie';
import FavoriteMovie from './FavoriteMovie';
import Form from './Form';
import { useTheme } from './useTheme';

const App = () => {
  const { theme, toggleTheme } = useTheme();
  const [favMovies, setFavMovies] = React.useState([]);

  const addMovie = (movie) => {
    setFavMovies([...favMovies, movie]);
  };

  return (
    <div className={`${theme} main`}>
      <Navbar toggleTheme={toggleTheme}/>
      <RandomMovie addMovie={addMovie}/>
      <FavoriteMovie favMovies={favMovies}/>
      <Form />
    </div>
  );
};

export default App;
