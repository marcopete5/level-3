Updating State in React: Ternary Operator

In this lesson, we explored how to manage and update state in React using the useState hook. We used a ternary operator to toggle between two states. Let’s break down how this works with the provided example code.

App Component

import { useState } from "react";
import RandomMovie from "./components/RandomMovie";
import Navbar from "./components/Navbar";

function App() {
  const [theme, setTheme] = useState('light');

  function toggleTheme() {
    setTheme(prevTheme => prevTheme === 'light' ? 'dark' : 'light');
  }

  return (
    <div className={`${theme} main`}>
      <Navbar toggleTheme={toggleTheme} />
      <RandomMovie />
    </div>
  );
}

export default App;

Navbar Component

export default function Navbar(props) {
  const { toggleTheme } = props;

  return (
    <nav className="nav">
      <h1>MovieBuddy</h1>
      <button onClick={toggleTheme}>Toggle Theme</button>
    </nav>
  );
}

Detailed Explanation

  1. Initializing State: 
    const [theme, setTheme] = useState('light');
    We initialize the theme state with an initial value of 'light'. setTheme updates the state.
  2. Toggling State with a Ternary Operator: 
    function toggleTheme() {
  setTheme(prevTheme => prevTheme === 'light' ? 'dark' : 'light');
}
    The updater uses a ternary to toggle between 'light' and 'dark' based on the previous state.
  3. Passing the Function as a Prop: 
    <Navbar toggleTheme={toggleTheme} />
    The toggleTheme function is passed to Navbar.
  4. Using the Function in the Navbar Component: 
    <button onClick={toggleTheme}>Toggle Theme</button>
    Clicking the button triggers the state toggle.
  5. Applying the Theme: 
    <div className={`${theme} main`}>
    The current theme is applied to the container for conditional styling.

Why Use a Ternary Operator?

The ternary operator is a concise way to handle conditional logic directly within a state updater, making toggle logic compact and readable.

Summary

  • Initialize state with useState.
  • Use a ternary in the updater to toggle values based on previous state.
  • Pass the updater as a prop to child components to handle interactions.
  • Apply conditional class names to reflect state in the UI.
