Updating State in React: Ternary Operator

In this lesson, we explore how to manage and update state in React using the useState hook. We use a ternary operator to toggle between two states. Let’s break down how this works with the provided example code.

The useState hook is essential for state management in functional components. The ternary operator enables concise state toggling, but improper use can cause issues. A common pitfall is neglecting the functional update form, risking stale state: 


/* Incorrect */
function toggleTheme() {
    setTheme(theme === 'light' ? 'dark' : 'light'); // Risks stale state
}

Using prevTheme => ensures the latest state, especially during rapid updates. Another pitfall is overusing ternaries for complex logic, which can obscure code. For multi-state scenarios, consider a state machine or object mapping.

App Component

Here's the App component where we implemented the state toggle functionality: 


import { useState } from "react";
import RandomMovie from "./components/RandomMovie";
import Navbar from "./components/Navbar";

function App() {
    const [theme, setTheme] = useState('light');

    function toggleTheme() {
        setTheme(prevTheme => prevTheme === 'light' ? 'dark' : 'light');
    }

    return (
        <div className={`${theme} main`}>
            <Navbar toggleTheme={toggleTheme} />
            <RandomMovie />
        </div>
    );
}

export default App;

A practical example is a notification toggle, illustrating state and props: 


import { useState } from "react";

function Notification() {
    const [isEnabled, setIsEnabled] = useState(true);

    const toggleNotification = () => {
        setIsEnabled(prev => !prev);
    };

    return (
        <div className={isEnabled ? 'enabled' : 'disabled'}>
            <button onClick={toggleNotification}>
                {isEnabled ? 'Disable' : 'Enable'} Notifications
            </button>
        </div>
    );
}

This component toggles notification visibility, similar to the theme toggle, and is reusable in settings panels, demonstrating state-driven UI updates.
