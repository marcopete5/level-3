Common className Errors and Debugging

Identify and fix common className errors in JSX to ensure proper styling and functionality.

Learning Objective

Students will recognize and resolve className-related errors in JSX.

Common Errors

  1. Using class Instead of className:
    • Symptom: No styling, or console error (e.g., “Invalid DOM property”).
    • Fix: Replace class with className.
    • Example (Wrong):
      • <div class="container">Hello</div>
    • Example (Correct):
      • <div className="container">Hello</div>
  2. Incorrect Case (e.g., classname):
    • Symptom: Styles not applied, no error.
    • Fix: Use className (camelCase).
    • Example (Wrong):
      • <div classname="container">Hello</div>
    • Example (Correct):
      • <div className="container">Hello</div>
  3. Missing or Incorrect CSS Import:
    • Symptom: Styles not applied, no errors.
    • Fix: Verify CSS import path.
    • Example (Wrong):
      • // Missing import
function App() {
  return <div className="container">Hello</div>;
}
    • Example (Correct):
      • import './styles.css';

function App() {
  return <div className="container">Hello</div>;
}

Debugging Tips

  • DevTools: Check Elements tab for class attributes.
  • Console: Look for syntax errors or warnings.
  • Linting: Use ESLint with React plugins to catch class.

Best Practices

  • Always use className in JSX.
  • Organize CSS in src (e.g., styles.css).
  • Test styles incrementally to catch issues early.
