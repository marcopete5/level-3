Applying CSS with className

Learn to style React components using className instead of class, including advanced styling techniques.

Learning Objective

Students will correctly use className to apply CSS styles in JSX.

Why className?

In HTML, class applies CSS styles. In JSX, className is used because class is a reserved JavaScript keyword, preventing conflicts.

Transpilation: Babel converts className to class in the DOM.

HTML vs. JSX

HTML Example:

<div class="container">
  <h1 class="title">Hello, World!</h1>
  <p class="description">This is an example of using class in HTML.</p>
</div>

JSX Example:

import React from 'react';
import './styles.css';

function App() {
  return (
    <div className="container">
      <h1 className="title">Hello, World!</h1>
      <p className="description">This is an example of using className in JSX.</p>
    </div>
  );
}
export default App;

Applying CSS

Create and import a CSS file:

/* src/styles.css */
.container {
  background: #e8ecef;
  padding: 20px;
  border-radius: 6px;
}
.title {
  color: #4a6fa5;
  font-size: 1.8em;
}
.description {
  font-style: italic;
  color: #333333;
}

Advanced Usage

Multiple Classes: Use template literals:

function App() {
  const isActive = true;
  return (
    <div className={`container ${isActive ? 'active' : ''}`}>
      <h1 className="title primary">Welcome!</h1>
    </div>
  );
}

Dynamic className: Apply classes conditionally:

function Button() {
  const isDisabled = false;
  return (
    <button className={isDisabled ? 'btn disabled' : 'btn'}>
      Click Me
    </button>
  );
}

Pitfalls

  • Using class: Causes syntax errors or no styling.
  • Case Sensitivity: className, not classname.
  • Import Errors: Wrong CSS path (e.g., ./Styles.css).
