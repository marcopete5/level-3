Understanding useEffect in React

APIs in React

Learning Objectives

A student will be able to:

  • Understand and explain the purpose of the useEffect hook in React for handling side effects in function components.
  • Differentiate between component mounting and component rendering, and describe how useEffect behaves in each scenario.
  • Implement the useEffect hook with and without a dependency array to control when the effect runs in a React component.

Introduction

In this lesson, we'll delve into the useEffect hook in React, focusing on how it functions when a component mounts and when it renders. We'll also cover the basic syntax of useEffect without going into detail on the dependency array.

What is useEffect?

useEffect is a hook that allows you to perform side effects in function components. Common use cases include data fetching, subscriptions, and manually changing the DOM. useEffect runs after the render phase, ensuring that the DOM has been updated before your effect executes.

Basic Syntax

Here's a basic example of useEffect:


import React, { useEffect } from 'react';

const ExampleComponent = () => {
  useEffect(() => {
    // This code runs when the component mounts and on every render.
    console.log('Component mounted or rendered')
  }, []);

  return (
    <div>
      <h1>Hello, useEffect!</h1>
    </div>
  );
};

export default ExampleComponent;

Component Mounting vs. Component Rendering

Component Mounting

When a component mounts, useEffect runs after the initial render. This is useful for operations you only need to perform once, such as fetching data or setting up a subscription. 


useEffect(() => {
  console.log('Component mounted');
}, []);

In this example, the effect only runs once when the component mounts.

Component Rendering

By default, useEffect runs after every render. However, since we're using an empty dependency array ([]), the effect in our example only runs on mount and unmount. If you omit the dependency array, the effect will run after every render.

Recap

  • useEffect allows you to perform side effects in function components.
  • It runs after the component mounts and after every render unless you provide a dependency array.
  • With an empty dependency array ([]), useEffect only runs on mount and unmount.

Conclusion

Understanding when and how useEffect runs is crucial for managing side effects in your React components. Use the empty dependency array to run effects only on mount and unmount, and stay tuned for the next lesson where we'll explore the dependency array in more detail.
