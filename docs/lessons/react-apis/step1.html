Introduction to API Calls in React

In this lesson, we're going to explore how to fetch data in React using axios. We'll cover handling responses of different data types and avoiding infinite loops.

Using Axios for API Calls

Why Axios?

Axios is a popular HTTP client that simplifies making requests to APIs. Compared to the built-in fetch method, axios offers features like interceptors, automatic JSON transformation, and better error handling.

Making a GET Request

axios.get('https://swapi.dev/api/people/1')
  .then(response => console.log(response))
  .catch(error => console.error(error));

Handling Complex Responses

axios.get('https://ddragon.leagueoflegends.com/cdn/13.24.1/data/en_US/champion.json')
  .then(response => {
    const champs = Object.values(response.data.data);
    console.log(champs);
  })
  .catch(error => console.error(error));

Using State to Store API Data

const [leagueChamps, setLeagueChamps] = useState([]);

The Problem of Infinite Loops

Placing an axios call directly in the component body and updating state will re-render the component, which runs the axios call again, causing an infinite loop. 

// Anti-pattern: triggers infinite re-renders
axios.get('https://ddragon.leagueoflegends.com/cdn/13.24.1/data/en_US/champion.json')
  .then(response => {
    const champs = Object.values(response.data.data);
    setLeagueChamps(champs); // state update in render path
  })
  .catch(error => console.error(error));
