Understanding the Dependency Array in useEffect

In this lesson, we'll explore the dependency array in React's useEffect hook. Using the provided code example, we'll explain how the dependency array controls when the effect runs, helping you avoid unnecessary re-renders and infinite loops.

The Code Example

Let's take a look at the provided code example to see useEffect and the dependency array in action: 

import { useState, useEffect } from 'react';
import axios from 'axios';

function App() {
  const [starWarsData, setStarWarsData] = useState([]);
  const [count, setCount] = useState(1);

  function incCount() {
    setCount(prevCount => prevCount + 1);
  }

  useEffect(() => {
    axios.get(`https://swapi.dev/api/people/${count}`)
      .then(res => setStarWarsData(res.data))
      .catch(err => console.log(err));
  }, [count]);

  console.log(starWarsData);

  return (
    <div className="container">
      <h1>{starWarsData.name}</h1>
      <h1>API CALLS</h1>
      <p>axios</p>
      <p>fetch</p>
      <p>the response</p>
      <p>infinite loops</p>
      <p>useEffect</p>
      <button onClick={incCount}>Click Me</button>
    </div>
  );
}

export default App;

What is the Dependency Array?

The dependency array is an optional second argument in the useEffect hook. It is an array of values that the effect depends on. When one of these values changes, the effect will re-run. 

useEffect(() => {
  // effect logic here
}, [someValue, anotherValue]);

How it Works in the Example

useEffect(() => {
  axios.get(`https://swapi.dev/api/people/${count}`)
    .then(res => setStarWarsData(res.data))
    .catch(err => console.log(err));
}, [count]);
  • Runs on initial mount.
  • Runs again whenever count changes.

Why Use a Dependency Array?

  • Preventing Infinite Loops: Without a dependency array, the effect runs after every render; if the effect sets state, you can create a render loop.
  • Optimizing Performance: Specify only the values that should trigger the effect to avoid unnecessary work.

Common Dependency Array Patterns

1) Empty Dependency Array ([]) — run once on mount 

useEffect(() => {
  console.log('Component mounted');
}, []);

2) Single Dependency

useEffect(() => {
  console.log('Count changed:', count);
}, [count]);

3) Multiple Dependencies

useEffect(() => {
  console.log('Count or otherState changed');
}, [count, otherState]);

Avoiding Common Pitfalls

  • Including Unnecessary Dependencies: Can cause extra re-renders. Include only what the effect actually uses.
  • Omitting Necessary Dependencies: Can lead to stale values or bugs. If an effect uses a value, it generally belongs in the array.

Recap

  • The dependency array controls when an effect runs.
  • Use [] to run once on mount; omit the array to run after every render.
  • List relevant dependencies to prevent infinite loops and optimize performance.
