Ternary Operator

Syntax:


condition ? <TrueJSXElement /> : <FalseJSXElement />

Functionality:

  • The ternary operator (? :) is used for inline conditional rendering.
  • If the condition is true, the element before : is rendered; if false, the element after : is rendered.
  • Useful for toggling between two elements.

Example:


<button onClick={toggleDetails}>
    {showDetails ? "Hide Details" : "Show Details"}
</button>

Ternaries are concise but can become unreadable if nested. A common pitfall is using ternaries for complex logic: 


/* Incorrect */
{state ? value1 ? 'Option1' : 'Option2' : value2 ? 'Option3' : 'Option4'} // Hard to read

Use functions or components for complex conditions. Another pitfall is neglecting parentheses, causing precedence issues: 


/* Incorrect */
{showDetails ? "Hide" : "Show" + " Details"} // Concatenates incorrectly

An accordion component shows ternary usage:


import { useState } from "react";

function Accordion() {
    const [isOpen, setIsOpen] = useState(false);

    return (
        <div className="accordion">
            <button onClick={() => setIsOpen(prev => !prev)}>
                {isOpen ? 'Collapse' : 'Expand'}
            </button>
            {isOpen && (
                <div className="content">
                    <p>This is the accordion content.</p>
                </div>
            )}
        </div>
    );
}

This accordion toggles content visibility, ideal for FAQs or menus, using both ternary and && operators.
