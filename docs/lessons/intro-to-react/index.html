Introduction to React: Why React?

Welcome to our introduction module on React, a JavaScript library for building dynamic user interfaces. This session will teach you why React is preferred over Vanilla JS and how its declarative approach simplifies web development.

Learning Objective

Students will be able to explain React’s declarative programming and its advantages over Vanilla JS.

What is Declarative Programming?

React lets you describe what your UI should look like, and it handles the how (DOM updates). This declarative approach contrasts with Vanilla JS’s imperative programming, where you manually update elements.

Analogy: Ordering a pizza (declarative: “I want a pepperoni pizza”) vs. cooking it step-by-step (imperative: “Mix dough, add sauce, sprinkle cheese…”).

Benefits

  • Simpler Code: Focus on the UI’s end state, not update logic.
  • Fewer Bugs: React’s automatic updates reduce errors from manual DOM changes.
  • Better Readability: Declarative code is clear and concise.

Example

React (declarative):

function WelcomeMessage() {
  return <h1>Welcome to React!</h1>;
}

Vanilla JS (imperative):

const h1 = document.createElement('h1');
h1.textContent = 'Welcome to React!';
document.body.appendChild(h1);

Why React? It simplifies development, reduces errors, and scales better for complex UIs like social media feeds or dashboards.
