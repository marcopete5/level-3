Composable Components

React builds UIs from small, reusable pieces called components. This section explores how components make your code modular and maintainable.

Learning Objective

Students will be able to explain what reusable components are and their benefits.

What is Composability?

Components are self-contained units with their own structure, style, and logic, like LEGO blocks. You can combine them to create complex UIs.

Benefits

  • Reusability: Use components (e.g., a button) across your app.
  • Maintainability: Small components are easier to test and debug.
  • Scalability: Modular code grows with your app’s complexity.

Example

A reusable Header component:

function Header() {
  return <h1>My Website</h1>;
}

function App() {
  return (
    <div>
      <Header />
      <p>Welcome to my website!</p>
    </div>
  );
}

Nested Components: Components can include others: 

function Button() {
  return <button>Click Me</button>;
}

function Card() {
  return (
    <div>
      <h2>Card Title</h2>
      <Button />
    </div>
  );
}

function App() {
  return (
    <div>
      <Header />
      <Card />
    </div>
  );
}

Common Pitfall

Ensure components are exported/imported correctly (e.g., export default Header;). Check DevTools for errors like “Component is not defined.”

In-Class Exercise (10 min): Sketch a UI (e.g., a blog) on paper and break it into 3-4 components (e.g., Header, Footer, Button). Share with the class.
