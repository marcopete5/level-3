Handling Input Changes in React Forms

When working with forms in React, one of the most common tasks is handling changes to input fields. This process ensures that the form data is kept in sync with the component's state. In the provided code, the handleChange function plays a crucial role in managing these updates efficiently.

Understanding handleChange

  1. Event Handling: The function receives an event object e as its parameter. This object contains information about the event that triggered the handler, including the target element (the input field).
  2. Destructuring: Inside the function, the name and value properties are extracted from the event target (e.target). The name corresponds to a key in state, and value is the current input value.
  3. State Update: setFormData updates state using the previous state to preserve other fields, and a computed key ([name]) to set the changed field.

Key Concepts in Handling Input Changes

  • Controlled Components: Inputs derive their value from state and update state on change.
  • Single Source of Truth: Component state holds the canonical form values.
  • Dynamic Keys: Use [name]: value to handle many inputs with one change handler.

Practical Example

import React from 'react';

export default function Form() {
  const [formData, setFormData] = React.useState({
    title: '',
    description: '',
    rating: ''
  });

  function handleChange(e) {
    console.log(e.target); // log the event target
    const { name, value } = e.target;
    setFormData(prevFormData => ({
      ...prevFormData,
      [name]: value
    }));
  }

  function handleSubmit(e) {
    e.preventDefault();
    console.log('Submitting:', formData);
  }

  return (
    <form onSubmit={handleSubmit}>
      <label htmlFor="title">Title</label>
      <input
        id="title"
        name="title"
        type="text"
        value={formData.title}
        onChange={handleChange}
        placeholder="Movie title"
      />

      <label htmlFor="description">Description</label>
      <textarea
        id="description"
        name="description"
        value={formData.description}
        onChange={handleChange}
        placeholder="Short description"
      />

      <label htmlFor="rating">Rating</label>
      <input
        id="rating"
        name="rating"
        type="number"
        min="0"
        max="10"
        step="0.1"
        value={formData.rating}
        onChange={handleChange}
        placeholder="e.g. 8.6"
      />

      <button className="btn-primary" type="submit">Save Movie</button>
    </form>
  );
}

Conclusion

Handling input changes in React forms is foundational for building interactive UIs. Using controlled components and a centralized state keeps your data consistent with user input. The handleChange pattern above scales well as forms grow.
