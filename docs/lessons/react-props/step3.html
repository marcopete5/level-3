Introduction to Data Mapping

Learn to use the .map method to render dynamic lists in React, separating data and UI logic.

Learning Objectives

  • Understand the purpose of .map in React.
  • Implement data mapping to generate JSX elements.

What is Data Mapping?

The .map method iterates over an array and returns a new array of JSX elements, ideal for rendering lists dynamically.

Analogy: .map is like a production line, transforming each item (data) into a finished product (JSX element).

Example: Rendering Skateboards

Map an array of skateboards to JSX elements:

const skateboards = [
  { id: 1, name: "Street Cruiser", price: 75.99 },
  { id: 2, name: "Pro Trickster", price: 89.99 }
];

const skateboardElements = skateboards.map(board => (
  <div className="skateboard">
    <h2>{board.name}</h2>
    <p>Price: ${board.price.toFixed(2)}</p>
  </div>
));

Why Use .map?

  • Dynamic Lists: Automatically render elements based on data.
  • Separation of Concerns: Keep data logic separate from UI.
  • Scalability: Easily handle growing datasets.

Rendering in a Component

Include mapped elements in a component:

function Store({ skateboards }) {
  return (
    <div>
      {skateboards.map(board => (
        <div className="skateboard">
          <h2>{board.name}</h2>
          <p>Price: ${board.price.toFixed(2)}</p>
        </div>
      ))}
    </div>
  );
}

Pitfall

Forgetting to return a single JSX element per .map iteration causes errors. Ensure each iteration returns one element.
