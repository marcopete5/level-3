What are Props?

Props (short for "properties") are a mechanism in React to pass data from a parent component to a child component, making components dynamic and reusable.

Analogy: Props are like ingredients passed to a recipe (component). The same recipe can produce different dishes based on the ingredients provided.

Key Features

  • Data Passing: Props allow parent components to send data (e.g., strings, arrays, objects) to children.
  • Read-Only: Props are immutable in the child component, ensuring predictable data flow.
  • Flexibility: Props enable the same component to render different content based on input.

Why Use Props?

  • Dynamic Rendering: Display varied content without duplicating code.
  • Reusability: Reuse components across the app with different data.
  • Separation of Concerns: Keep components focused on UI, not data management.

Example

A simple component using props for a skateboard shop:

function Navbar(props) {
  return <h1>Welcome to {props.name}</h1>;
}

function App() {
  return <Navbar name="SkateShop" />;
}
export default App;

Pitfall

Props are read-only. Attempting to modify props in a child component (e.g., props.name = "NewName") will cause errors.
