Using Props for Dynamic Rendering

Learn to pass props from parent to child components, render dynamic content, and use destructuring for cleaner code.

Learning Objectives

  • Pass and use props in components.
  • Implement dynamic rendering with props.

Passing Props

In a parent component, pass props as attributes to a child component. The child receives them as an object.

Example: Skateboard Shop

Pass name and skateboards props to a Store component: 

function App() {
  const storeName = "SkateShop";
  const skateboards = [
    { id: 1, name: "Street Cruiser", price: 75.99 },
    { id: 2, name: "Pro Trickster", price: 89.99 }
  ];
  return <Store name={storeName} skateboards={skateboards} />;
}

function Store(props) {
  return (
    <div>
      <h1>{props.name}</h1>
      <p>Items: {props.skateboards.length}</p>
    </div>
  );
}
export default App;

Destructuring Props

Destructure props in the child component for cleaner code:

function Store({ name, skateboards }) {
  return (
    <div>
      <h1>{name}</h1>
      <p>Items: {skateboards.length}</p>
    </div>
  );
}

Dynamic Rendering

Use props to render varied content. Example with conditional rendering: 

function Skateboard({ name, price, inStock }) {
  return (
    <div className="skateboard">
      <h2>{name}</h2>
      <p>Price: ${price.toFixed(2)}</p>
      <p>{inStock ? "In Stock" : "Out of Stock"}</p>
    </div>
  );
}

Pitfalls

  • Missing Props: Accessing undefined props (e.g., props.unknown) causes errors. Use default values or checks.
  • Incorrect Destructuring: Ensure prop names match (e.g., {name}, not {Name}).
