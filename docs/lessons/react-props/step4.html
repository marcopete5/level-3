Using Keys in Mapping

Learn the importance of the key prop in React lists and how it ensures efficient rendering.

Learning Objective

  • Implement the key prop when mapping arrays.
  • Understand the role of key in efficient updates.

Why Use the key Prop?

The key prop is a unique identifier for each element in a mapped list, helping React efficiently add, remove, or update elements.

Analogy: Keys are like name tags at a party, letting React quickly identify and update specific guests (elements).

How Keys Work

  • Adding: New elements are inserted without re-rendering the entire list.
  • Removing: React removes only the element with the matching key.
  • Updating: Only the changed element is updated.

Example: Skateboard List

Add key props using id:

function Store({ skateboards }) {
  return (
    <div>
      {skateboards.map(board => (
        <div className="skateboard" key={board.id}>
          <h2>{board.name}</h2>
          <p>Price: ${board.price.toFixed(2)}</p>
        </div>
      ))}
    </div>
  );
}

Pitfalls

  • Missing Keys: Causes React to re-render unnecessarily, leading to performance issues.
  • Non-Unique Keys: Using array indices can cause bugs if the list order changes.
