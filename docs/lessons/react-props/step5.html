Handling Nested Data

Learn to map nested data structures, such as arrays within objects, and apply key props for efficient rendering.

Learning Objective

  • Handle nested data with .map and unique key props.

Nested Data in React

Objects often contain arrays (e.g., a skateboard’s sizes). Use nested .map calls to render these arrays, ensuring each level has unique keys.

Example: Skateboard Sizes

Render a skateboard’s sizes array:

function Skateboard({ name, size, style, design, price }) {
  const sizeElements = size.map((num, index) => (
    <p key={index}>Size: {num}</p>
  ));
  return (
    <div className="skateboard">
      <h2>{name}</h2>
      {sizeElements}
      <p>Style: {style}</p>
      <p>Design: {design}</p>
      <p>Price: ${price.toFixed(2)}</p>
    </div>
  );
}

Data Structure

Example skateboard object with nested sizes:

{
  id: 1,
  name: "Street Cruiser",
  size: ["8.0", "6.0"],
  style: "Street",
  design: "Classic Black",
  price: 75.99
}

Best Practices

  • Unique Keys: Use indices for nested arrays if no unique ID exists, but prefer IDs for top-level lists.
  • Clear Structure: Keep nested .map calls readable with consistent indentation.

Pitfall

Using indices as keys for dynamic lists (e.g., reorderable lists) can cause rendering bugs. Always use unique IDs when available.
