Header, Navbar, and Footer Components

Build header, navbar, and footer components for a website layout, emphasizing separate component functions.

Learning Objectives

Students will create header, navbar, and footer components and understand the need for separate functions.

Why Separate Components?

  • Modularity: Each component handles a specific UI part, enhancing reusability.
  • Collaboration: Teams can work on different components simultaneously.
  • Maintenance: Update one component without affecting others.

Component Structure

Organize in src/components:

src/
├── components/
│   ├── Header.jsx
│   ├── Navbar.jsx
│   ├── Footer.jsx
├── App.jsx
└── main.jsx

Header Component

// src/components/Header.jsx
function Header() {
  return (
    <header style={{ background: "#4a6fa5", color: "#ffffff", padding: "10px", textAlign: "center" }}>
      <h1>My Website</h1>
    </header>
  );
}
export default Header;

Navbar Component

// src/components/Navbar.jsx
function Navbar() {
  return (
    <nav style={{ background: "#e8ecef", padding: "10px" }}>
      <ul style={{ listStyle: "none", display: "flex", justifyContent: "center", gap: "20px" }}>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
        <li><a href="#">Contact</a></li>
      </ul>
    </nav>
  );
}
export default Navbar;

Footer Component

// src/components/Footer.jsx
function Footer() {
  return (
    <footer style={{ background: "#2f2f2f", color: "#ffffff", padding: "10px", textAlign: "center" }}>
      <p>© 2025 My Website. All rights reserved.</p>
    </footer>
  );
}
export default Footer;

Using in App

// src/App.jsx
import Header from './components/Header';
import Navbar from './components/Navbar';
import Footer from './components/Footer';

function App() {
  return (
    <>
      <Header />
      <Navbar />
      <main>
        <h2>Welcome</h2>
        <p>This is the main content.</p>
      </main>
      <Footer />
    </>
  );
}
export default App;

Pitfalls

  • File Paths: Incorrect imports (e.g., ./Components/Header) cause errors.
  • Styling: Inline styles may conflict; consider CSS files for complex designs.
