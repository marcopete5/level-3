React Fragments

Explore React fragments, a tool for grouping JSX elements without adding extra DOM nodes.

Learning Objective

Students will describe React fragments and their purpose.

What is a React Fragment?

Fragments (<></> or <React.Fragment></React.Fragment>) group multiple JSX elements without adding a DOM element (e.g., <div>), keeping HTML clean.

Why Use Fragments?

  • Cleaner DOM: Avoid unnecessary wrappers that bloat HTML.
  • Better Styling: Prevent styling issues from extra <div>s.
  • Semantic HTML: Maintain accessibility-friendly structure.

Example: Without Fragment

function App() {
  return (
    <div>
      <h1>Why Learn React?</h1>
      <ol>
        <li>Popular library</li>
        <li>Better job prospects</li>
      </ol>
    </div>
  );
}

This adds an extra <div>, which may affect styling or semantics.

Example: With Fragment

function App() {
  return (
    <>
      <h1>Why Learn React?</h1>
      <ol>
        <li>Popular library</li>
        <li>Better job prospects</li>
      </ol>
    </>
  );
}

The fragment groups elements without a DOM node.

Explicit Fragment

Use <React.Fragment> for clarity or with key attributes in lists: 

function ItemList() {
  const items = ["Apple", "Banana"];
  return (
    <React.Fragment>
      {items.map(item => (
        <React.Fragment key={item}>
          <li>{item}</li>
        </React.Fragment>
      ))}
    </React.Fragment>
  );
}

Pitfalls

  • Overuse: Use fragments only when avoiding extra nodes is needed.
